## 前言

Perplexity 宣布其 Personal Computer 功能正式向所有 Mac 用户开放，无需再加入等待名单。这一动作意味着，AI Agent 从云端向本地设备迁移的趋势进一步加速。Perplexity 将 Personal Computer 定位为 OpenClaw 和其他本地 AI Agent 的「更安全替代方案」，强调其不要求用户赋予 Agent 提升的系统权限。

## 什么是 Perplexity Personal Computer

Perplexity Personal Computer 是该公司在今年 4 月推出的一项功能，它将 Perplexity 的多模型 AI 工作能力从云端扩展到用户的 Mac 设备。与传统的通用型聊天 AI 不同，Personal Computer 允许 AI Agent 访问用户的本地文件、应用程序，并能与超过 400 个外部连接器协作，完成多步骤的个人工作流。

值得注意的概念区分：Perplexity 此前还有一个叫「Perplexity Computer」的产品，它是基于云端的多模型数字助手。而 Personal Computer 则旨在将这种能力带到用户自己的设备上，用 Perplexity 的话说，就是「将 Computer 从纯云端世界带到大多数真实工作发生的设备上」。

## 全面开放意味着什么

Personal Computer 在 4 月推出时，仅面向 Perplexity Max 订阅用户且需加入等待名单。5 月 7 日的更新后，所有 Mac 用户均可下载全新的 Perplexity Mac 应用并使用该功能——不过 Personal Computer 本身仍需要 Pro 或 Max 订阅。

新版 Mac 应用的核心交互方式是「Command Bar」：用户同时按下两个 Command 键即可呼出。它支持以下能力：

- **本地文件操作**：直接读取和编辑 Mac 上的文件
- **原生 Mac 应用协作**：跨应用操作，例如从一个应用提取内容到另一个应用
- **网络工具编排**：配合 Perplexity AI 浏览器 Comet，可操控网页端工具
- **远程访问**：即便不在电脑旁，用户也可以通过 iPhone 发起任务或审批请求
- **安全开发环境**：核心处理在 Perplexity 服务器的安全沙箱中运行，而非直接在用户设备上以提升权限运行

Perplexity 还宣布，旧版 Mac 应用将在数周内停用，团队将把所有资源集中在 Personal Computer 上。

## 与 OpenClaw 的安全对比

Perplexity 主动在公告中将 Personal Computer 与 OpenClaw 进行了安全对比。今年 4 月，OpenClaw 因安全风险受到业界质疑，Cisco 和多家安全机构指出其在提升权限下运行的架构存在系统性风险。

Perplexity 的逻辑是：通过将 AI 处理逻辑放在云端安全沙箱而非本地提升权限运行，可以规避这个问题。当然这也意味着用户的数据仍然经过 Perplexity 的服务器——隐私权衡由用户自行判断。

## 我们的判断

**对普通 Mac 用户**：如果你一直在等待一个能真正帮你操作文件和多应用的本地 AI，而不想冒险给 Agent 系统级权限，Personal Computer 是目前最低门槛的选项。Pro 订阅（月费约 20 美元）即可获得，配置为零。

**对 AI Agent 开发者**：Perplexity 的「安全替代」叙事值得跟踪。如果这个定位被市场接受，可能会影响企业部署本地 Agent 的安全标准走向。

**对 OpenClaw 用户**：Perplexity 的强势入局意味着竞争加剧。如果你是 OpenClaw 重度用户，建议关注 Perplexity 后续在 Windows 和企业场景的扩展计划。
