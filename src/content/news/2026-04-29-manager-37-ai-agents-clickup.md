## 前言

不是所有关于 AI Agent 的深刻洞察都来自论文或技术大会。有时候，最真实的变化发生在某个增长负责人悄悄建起第 37 个 Agent 的那个下午。

Jordan Meyrowitz 是 ClickUp 增长运营团队的负责人。2026 年 4 月，他在 Business Insider 的一篇采访中分享了自己的故事：一年前，他所在的公司被 ClickUp 收购，他转型负责增长运营——一个本质上是「协调大量碎片化任务」的角色。一年后，他管理着 **37 个 AI Agent**，这些 Agent 帮他处理日程安排、数据分析和会议跟进，而他自己的工作时间结构发生了根本性变化。

这个故事值得认真拆解，因为它代表了一种正在扩散但极少被公开记录的职场现实。

## 37 个 Agent 都在做什么

Jordan 并没有给出一个完整的 Agent 清单，但采访透露了几个典型场景：

**日程管理 Agent**：帮他协调跨时区会议，自动判断参会人的最佳可用时间，并在会议冲突时主动提议调整方案。

**数据分析 Agent**：从 CRM 和营销平台拉取数据，生成周报摘要，并在他追问「为什么上周转化率下降了 3%」时，主动调取同期投放素材和外部事件进行相关性分析。

**会议跟进 Agent**：在会议结束后自动提取行动项，向相关同事发送 follow-up 邮件，并在截止日期前 24 小时发出提醒。

**竞品监控 Agent**：持续追踪 8 家直接竞品的产品更新、融资动态和招聘信息，每天早上推送简报。

这些场景并不新鲜——任何一个用过 Zapier、Make 或 ChatGPT Agents 的人都听过类似的能力列表。但 Jordan 的案例有意思的地方在于：**他没有任何编程背景**，这些 Agent 的部署、维护和优化，全部靠他一个人完成。

## 从「管人」到「管 Agent」：能力模型的根本转变

Jordan 故事中最反直觉的一点是：管理 37 个 Agent 并不需要技术背景，但它确实要求一套与传统管理截然不同的能力模型。

**传统增长负责人的核心能力**：沟通、项目管理、行业知识、数据解读。

**管理 37 个 Agent 所需的额外能力**：

- **工作流解构能力**：把一个业务目标拆解成 Agent 能执行的有限步骤
- **Prompt 工程直觉**：知道什么样的指令格式能让 Agent 输出稳定可用的结果
- **跨 Agent 协调思维**：当一个任务的输出成为另一个 Agent 的输入时，如何设计握手协议
- **边界判断力**：知道什么任务适合交给 Agent，什么任务必须保留给人

Jordan 在采访中提了一个很精准的描述：「我现在更像是一个小型工厂的厂长，而不是一个个人贡献者。我不需要亲手做每件事，但我需要知道每个工位的工人（Agent）什么时候会出错，以及怎么调整流水线。」

这个比喻值得所有 AI Agent 产品的设计者认真思考：**如果用户需要「当厂长」，你的产品是否给了他们足够的管理工具？**

## Agent 管理的真实痛点：不是技术，是「可见性」

在 Jordan 的描述中，最让他头疼的并不是 Agent 不会做事，而是**不知道 Agent 在做什么**。

当 Agent 数量从 5 个增加到 20 个，再到 37 个，他面临的核心问题是：如何在不逐个检查的情况下，确保所有 Agent 都在正确执行任务？

他采取的办法是建立一套「异常上报机制」：每个 Agent 被设定为在遇到无法处理的情况或结果偏离预期时，主动发送通知。没有异常，则默认一切正常。

这本质上是一种**面向 Agent 的告警系统设计**——而这类工具在整个 Agent 生态中仍然相当初级。

## 人机协作的边界：什么被改变了，什么没有

Jordan 特别提到，他的角色并没有因此变成「闲人」——相反，他花在「审核 Agent 输出」上的时间远大于他花在「亲自做任务」上的时间。他说：「以前我是执行者，现在我是裁判。但这个裁判需要懂规则。」

被彻底改变的：日程协调、数据汇总类、竞品监控、邮件模板生成。
仍然需要人的：与外部合作方的谈判、需要判断力的战略决策、跨部门利益协调。

**一个值得注意的观察**：当 Agent 接管了大量「执行层」任务后，Jordan 反而觉得自己更忙了——因为执行层任务释放出来后，他被邀请参与的决策会议反而更多了。「以前我负责执行战略，现在战略讨论也拉上我了。」

## 我们的判断

Jordan Meyrowitz 的案例代表了一种正在发生的职业形态转变：不是「AI 取代人」，而是「会用 Agent 的人取代不会用 Agent 的人」，而且这个取代的速度比多数人预期的更快。

**对哪类读者有用：**
- 正在考虑引入 AI Agent 提升团队效率的运营和产品负责人
- 对 AI Agent 在实际职场落地效果存疑的技术决策者
- 正在探索「第二职业曲线」的知识工作者

有一点需要特别指出：Jordan 的成功有其特定条件——他所在的公司被 ClickUp 收购，天然对 Agent 产品有更高接受度，且他本人对效率工具极度敏感。复制这个模式需要匹配的组织土壤，而非简单的「我也部署 37 个 Agent」。
